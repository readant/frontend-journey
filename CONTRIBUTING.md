# 贡献指南

感谢你对 Frontend Journey 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果你发现了 bug 或有改进建议：

1. 检查 [Issues](https://github.com/your-username/frontend-journey/issues) 看是否已有相关问题
2. 如果没有，创建一个新的 Issue，包含：
   - 清晰的问题描述
   - 复现步骤（如果适用）
   - 期望的行为
   - 实际的行为
   - 截图（如果适用）

### 提交代码

1. **Fork 本仓库**
2. **创建特性分支**
   ```bash
   git checkout -b feature/你的特性名称
   ```
3. **提交更改**
   ```bash
   git commit -m "添加: 简短描述你的更改"
   ```
   提交信息格式建议：
   - `添加:` 新功能
   - `修复:` bug 修复
   - `更新:` 改进现有功能
   - `文档:` 文档更新
   - `样式:` 代码格式调整（不影响功能）

4. **推送到你的 Fork**
   ```bash
   git push origin feature/你的特性名称
   ```

5. **创建 Pull Request**
   - 提供清晰的标题和描述
   - 引用相关的 Issue（如果有）
   - 说明你的更改内容和原因

## 贡献类型

我们欢迎以下类型的贡献：

### 内容贡献

- 新的学习笔记和教程
- 改进现有文档
- 添加代码示例和练习
- 翻译内容

### 代码贡献

- 修复错误
- 改进现有示例
- 添加新的项目示例
- 优化性能

### 社区贡献

- 回答问题
- 审查 Pull Request
- 改进项目文档
- 分享学习资源

## 贡献原则

1. **保持简洁** — 代码和文档都应该清晰易懂
2. **保持一致** — 遵循项目的编码风格和目录结构
3. **保持质量** — 确保你的贡献是有价值的
4. **保持友好** — 尊重其他贡献者

## 编码规范

### HTML

- 使用语义化标签
- 保持良好的缩进（2 个空格）
- 添加必要的注释

### CSS

- 使用有意义的类名
- 保持选择器简洁
- 遵循 BEM 命名规范（可选）

### JavaScript

- 使用 ES6+ 语法
- 保持函数简洁
- 添加必要的注释

## 工作流程

1. 选择一个 Issue 或创建一个新的
2. 声明你要处理这个 Issue（避免重复工作）
3. 创建分支并开始工作
4. 定期提交更改
5. 创建 Pull Request
6. 等待审查和反馈
7. 根据反馈进行修改
8. 合并到主分支
