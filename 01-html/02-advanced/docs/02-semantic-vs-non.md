# 语义化 vs 非语义化对比分析

> 对应 HTML 文件：`../02-semantic-vs-non.html`

## 本案例学习目标

- 理解语义化与非语义化的代码差异
- 理解语义化对 SEO、无障碍、可维护性的影响
- 能在实际项目中做出正确的标签选择

## 知识点详解

### 1. 两种写法对比

#### 非语义化写法（不推荐）

```html
<div class="header">
  <div class="nav">...</div>
</div>
<div class="main">
  <div class="article">
    <div class="section">...</div>
  </div>
</div>
<div class="sidebar">...</div>
<div class="footer">...</div>
```

**问题：**
- 只有开发者知道 div 的含义
- 搜索引擎难以理解
- 屏幕阅读器无法识别
- 代码可读性差

#### 语义化写法（推荐）

```html
<header>
  <nav>...</nav>
</header>
<main>
  <article>
    <section>...</section>
  </article>
</main>
<aside>...</aside>
<footer>...</footer>
```

**优点：**
- 标签本身就表达了含义
- SEO 更友好
- 无障碍访问更好
- 代码可读性强

### 2. 为什么语义化很重要？

#### 搜索引擎爬虫视角

搜索引擎通过标签结构理解页面内容：

| 标签 | SEO 影响 |
|------|----------|
| `<main>` | 内容权重最高，搜索引擎重点索引 |
| `<article>` | 内容可被单独索引（如 Google 带摘要的搜索结果） |
| `<h1>`~`<h6>` | 构成页面大纲，影响关键词排名 |
| `<nav>` | 帮助搜索引擎理解网站结构 |
| `<div>` | 无语义，搜索引擎不知道什么是重点 |

**实际案例：**
- 使用 `<article>` 包裹博客文章 → Google 可能显示文章摘要
- 使用 `<h1>`~`<h6>` 构建标题层级 → 搜索引擎更好理解内容结构
- 使用 `<nav>` 包裹导航 → 搜索引擎能识别网站导航结构

#### 屏幕阅读器视角

视障用户使用屏幕阅读器浏览网页：

| 标签 | 屏幕阅读器行为 |
|------|----------------|
| `<nav>` | 提供"跳转到导航"快捷键 |
| `<main>` | 提供"跳转到主要内容"快捷键 |
| `<article>` | 朗读为"文章"，可以单独跳过 |
| `<aside>` | 朗读为"侧边栏"，可以跳过 |
| `<div>` | 无法提供任何导航辅助 |

**实际案例：**
- 有 nav → 用户可以快速跳过导航，直接读内容
- 有 main → 用户可以跳过侧边栏，直接读核心内容
- 全是 div → 用户必须从头听到尾，浪费大量时间

#### 代码可维护性视角

团队协作时：

| 写法 | 可维护性 |
|------|----------|
| `<header>` + `<nav>` | 一眼看出导航在哪 |
| `<div class="header">` + `<div class="nav">` | 需要逐个看 class 才知道结构 |
| `<article>` | 知道这是一篇独立文章 |
| `<div class="article">` | 需要看 CSS 才知道这个 div 干什么 |

### 3. 语义化决策流程

```
需要包裹内容？
├── 能独立分发/阅读？ → article
├── 有独立主题？ → section
├── 是导航区域？ → nav
├── 是页头区域？ → header
├── 是页脚区域？ → footer
├── 是补充内容？ → aside
└── 都不是 → div
```

### 4. 常见误区

| 误区 | 正确做法 |
|------|----------|
| 所有内容都用 div 包裹 | 优先使用语义标签，实在没有合适标签时再用 div |
| main 放多个 | 一个页面只放一个 `<main>` |
| section 当 div 用 | section 需要有独立主题，应包含标题（h2-h6） |
| article 只能放文章 | article 可以是任何可独立分发的内容：评论、卡片、widget |
| aside 只能放右侧 | aside 可以放任意位置，关键是"与主内容相关但可独立" |
| 语义标签有默认样式 | 语义标签默认和 div 一样是块级元素，需要 CSS 来定义视觉效果 |

### 5. 实际项目中的应用

```html
<!-- 电商网站商品列表 -->
<main>
  <section>
    <h2>热门商品</h2>
    <article class="product-card">
      <img src="product.jpg" alt="商品图片">
      <h3>商品名称</h3>
      <p>价格：¥99</p>
    </article>
    <article class="product-card">...</article>
  </section>
</main>

<aside>
  <h3>购物车</h3>
  <p>共 3 件商品</p>
</aside>
```

```html
<!-- 新闻网站 -->
<main>
  <article>
    <header>
      <h1>新闻标题</h1>
      <time datetime="2026-07-14">2026-07-14</time>
      <address>记者：L君</address>
    </header>
    <section>
      <h2>正文</h2>
      <p>新闻内容...</p>
    </section>
    <footer>
      <p>来源：新华社</p>
    </footer>
  </article>
</main>
```

## 常见报错与踩坑

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 语义标签没样式 | 语义标签默认无样式 | 用 CSS 定义视觉效果 |
| section 没标题 | section 应有 h2-h6 | 加标题或改用 div |
| main 嵌套在 article 内 | main 不能嵌套在其他标签内 | 移到最外层 |
| 滥用 article | 不是所有内容都是 article | 只有能独立分发的才用 article |

## 本案例知识速查表

| 对比项 | 非语义化 | 语义化 |
|--------|----------|--------|
| 代码可读性 | 需要看 class | 标签自解释 |
| SEO | 搜索引擎难理解 | 搜索引擎易理解 |
| 无障碍 | 阅读器无法导航 | 阅读器可快速跳转 |
| 可维护性 | 团队协作困难 | 团队协作顺畅 |
| 推荐度 | ❌ 不推荐 | ✅ 推荐 |
