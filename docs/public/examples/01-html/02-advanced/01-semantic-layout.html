<header> 页面头部

这是页面的顶部区域，通常包含Logo、标题、导航等

一个页面可以有多个header，但最好有一个主导航的header

<nav> 导航栏

  • 首页
  • 文章
  • 关于
  • 联系

<main> 主要内容区

这里是页面的核心内容，一个页面应该只有一个main标签

<article> 独立文章

article标签表示一个完整的、可以独立阅读的内容块

比如：一篇博客文章、论坛帖子、新闻报道等

<section> 内容分区 1

section表示有独立主题的内容区域

每个section通常包含一个h2-h6标题

<section> 内容分区 2

这是另一个内容分区，有自己的主题

<aside> 侧边栏

aside包含与主要内容相关但又可以独立的内容

  • 相关链接
  • 热门文章
  • 广告位
  • 作者信息

<time> 日期时间标签

文章发布于 2026年7月14日

事件时间 圣诞节晚上6点

datetime 属性放机器可读的值，标签内放人类可读的显示文本

<address> 联系信息

作者：L君

邮箱：example@test.com

博客：我的技术博客

address 通常放在 header 或 footer 内，表示作者/联系人信息

role 属性（语义化补充）

当没有合适的 HTML5 语义标签时，可用 role 属性增强语义：

role="banner" = 页面横幅（类似 header）
role="main" = 主要内容（类似 main）
role="complementary" = 补充内容（类似 aside）
role="contentinfo" = 页脚信息（类似 footer）

WAI-ARIA role 用于增强无障碍访问，优先使用原生 HTML5 标签

<footer> 页脚

这是页面的底部区域，通常包含版权信息、联系方式等
