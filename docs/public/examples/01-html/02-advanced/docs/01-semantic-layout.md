# HTML5 语义化标签详解

> 对应 HTML 文件：`../01-semantic-layout.html`

## 本案例目标

- 掌握每个语义标签的确切含义和使用场景
- 理解标签之间的嵌套关系
- 能用语义标签构建完整页面

---

## 标签详解

### 1. `<header>` — 页面头部

```html
<header>
  <h1>网站标题</h1>
  <nav>...</nav>
</header>
```

**要点：**
- 可包含：标题（h1-h6）、导航（nav）、搜索框、Logo
- **可以有多个**：页面级一个 + 每个 article 内部也可以有
- 不一定在页面顶部，article 内部也可以有 header

---

### 2. `<nav>` — 导航区域

```html
<nav aria-label="主导航">
  <ul>
    <li><a href="/">首页</a></li>
    <li><a href="/blog">博客</a></li>
  </ul>
</nav>
```

**要点：**
- 用于**主要导航**，不是所有链接组都需要 nav
- 页脚的链接组也可以用 nav
- 用 `aria-label` 区分多个 nav（如"主导航"、"页脚导航"）

---

### 3. `<main>` — 主要内容

```html
<main>
  <!-- 页面的核心内容 -->
</main>
```

**关键规则：**
- **一个页面只能有一个**
- 不能嵌套在 article、aside、header、footer、nav 内
- 浏览器默认 role="main"，不需要额外加 role

---

### 4. `<article>` — 独立内容

```html
<article>
  <header>
    <h2>文章标题</h2>
    <time datetime="2026-07-14">2026-07-14</time>
  </header>
  <p>文章正文...</p>
</article>
```

**判断标准：这个内容能否被独立分发？**
- 能被 RSS 订阅 → article
- 能被单独转发/分享 → article
- 能被搜索引擎单独索引 → article

**典型场景：** 博客文章、新闻报道、论坛帖子、商品卡片、用户评论

---

### 5. `<section>` — 内容分区

```html
<section>
  <h2>第一章：入门</h2>
  <p>内容...</p>
</section>
```

**要点：**
- 表示有**独立主题**的内容区域
- 通常需要包含标题（h2-h6）
- 不要当 div 用！

---

### 6. `<aside>` — 侧边栏

```html
<aside aria-label="相关文章">
  <h3>相关推荐</h3>
  <ul>
    <li><a href="#">文章1</a></li>
  </ul>
</aside>
```

**要点：**
- 与主内容**相关但可独立**的内容
- **不一定在页面右侧**，可以放在任何位置
- 与 main 同级，不是嵌套在 main 内部

---

### 7. `<time>` — 日期时间

```html
<!-- 人类可读 + 机器可读 -->
<time datetime="2026-07-14">2026年7月14日</time>

<!-- 带时间 -->
<time datetime="2026-07-14T18:00">今晚6点</time>
```

**要点：**
- `datetime` 属性放机器可读的 ISO 8601 格式
- 标签内容放人类可读的显示文本
- 搜索引擎能提取结构化日期数据

---

### 8. `<address>` — 联系信息

```html
<address>
  写于 <a href="https://example.com">我的博客</a>。
  联系 <a href="mailto:me@example.com">我</a>。
</address>
```

**注意：** address 不是"地址"的意思，是"联系信息"。通常放在 header 或 footer 内。

关键规则：
- main 只能一个，不能嵌套在其他语义标签内
- header/footer/article 可以嵌套使用
- aside 与 main 同级，不在 main 内部

## section vs article 深度对比

这是最容易搞混的地方，必须理解清楚：

| 对比项 | `<section>` | `<article>` |
|--------|-------------|-------------|
| **语义** | 有独立主题的分区 | 可独立分发的内容 |
| **判断标准** | 有标题（h2-h6） | 能被 RSS/转发/单独索引 |
| **典型场景** | 文章的"引言"、"正文"、"结论" | 博客文章、新闻、评论 |
| **能否嵌套** | 可以有多个并列 | 可以有多个并列 |
| **内部结构** | 通常包含标题 + 内容 | 可以包含 section、header、footer |

**实际例子：**

```html
<!-- 博客首页：多篇文章 -->
<main>
  <article>  <!-- 第一篇文章 -->
    <h2>如何学习 JavaScript</h2>
    <section>
      <h3>第一步：基础语法</h3>
      <p>...</p>
    </section>
    <section>
      <h3>第二步：DOM 操作</h3>
      <p>...</p>
    </section>
  </article>
  
  <article>  <!-- 第二篇文章 -->
    <h2>CSS 布局技巧</h2>
    <p>...</p>
  </article>
</main>

<!-- 新闻详情页：一篇文章 -->
<article>
  <header>
    <h1>重大新闻标题</h1>
    <time datetime="2026-07-14">2026-07-14</time>
  </header>
  <section>
    <h2>事件经过</h2>
    <p>...</p>
  </section>
  <section>
    <h2>专家解读</h2>
    <p>...</p>
  </section>
  <footer>
    <address>记者：张三</address>
  </footer>
</article>
```

## 决策流程图

```
需要包裹内容？
│
├─ 能被独立分发（RSS/转发/单独索引）？
│  └─ 是 → <article>
│
├─ 有独立主题（需要标题 h2-h6）？
│  └─ 是 → <section>
│
├─ 是主要导航区域？
│  └─ 是 → <nav>
│
├─ 是页面/章节头部？
│  └─ 是 → <header>
│
├─ 是页面/章节底部？
│  └─ 是 → <footer>
│
├─ 是补充内容（相关但独立）？
│  └─ 是 → <aside>
│
└─ 都不是 → <div>
```

## 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| section 没有标题 | 语义不明确 | 加 h2-h6 标题，或改用 div |
| 页面有多个 main | 结构混乱 | 一个页面只放一个 main |
| aside 放在 main 内 | 语义错误 | 移到 main 外，与 main 同级 |
| 所有内容都用 div | 搜索引擎和阅读器无法理解 | 优先使用语义标签 |
| article 只放文章 | 限制了 article 的用途 | 任何可独立分发的内容都可以用 article |

## 拓展练习

1. **练习 1**：用语义标签重构一个只用 div 的页面
   - 提示：识别每个 div 的含义，替换为合适的语义标签

2. **练习 2**：创建一个博客文章页面
   - 要求：包含 article、section、aside、time、address
   - article 内部用 section 细分文章结构

3. **练习 3**：用 Chrome DevTools 检查页面结构
   - 打开任意网站 → Elements → Accessibility 面板
   - 观察语义标签如何被识别
