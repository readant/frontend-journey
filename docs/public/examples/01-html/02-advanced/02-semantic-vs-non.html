语义化 vs 非语义化标签对比

❌ 非语义化写法（只用 div）

<div class="header">
    <div class="nav">...</div>
</div>
<div class="main">
    <div class="article">
        <div class="section">...</div>
    </div>
</div>
<div class="sidebar">...</div>
<div class="footer">...</div>

问题：

  • 只有开发者知道 div 的含义
  • 搜索引擎难以理解
  • 屏幕阅读器无法识别
  • 代码可读性差

✅ 语义化写法（使用语义标签）

<header>
    <nav>...</nav>
</header>
<main>
    <article>
        <section>...</section>
    </article>
</main>
<aside>...</aside>
<footer>...</footer>

优点：

  • 标签本身就表达了含义
  • SEO更友好
  • 无障碍访问更好
  • 代码可读性强

为什么语义化很重要？

搜索引擎爬虫视角

搜索引擎通过标签结构理解页面内容：

  • <main> 内的内容权重最高
  • <article> 内容可被单独索引（如 Google 带摘要的搜索结果）
  • <h1>~<h6> 构成页面大纲，影响 SEO 排名
  • 全是 <div> = 搜索引擎不知道什么是重点

屏幕阅读器视角

视障用户使用屏幕阅读器浏览网页：

  • 语义标签让阅读器能提供"页面导航"（跳到 main、跳到 nav）
  • <nav> 让阅读器知道"这里是导航，可以跳过"
  • <article> 让阅读器知道"这是一篇独立文章"
  • 全是 <div> = 阅读器无法提供任何导航辅助

代码可维护性视角

团队协作时：

  • <header> + <nav> = 一眼看出导航在哪
  • <div class="header"> + <div class="nav"> = 需要逐个看 class 才知道结构
